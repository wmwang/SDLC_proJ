## 新增需求（ADDED Requirements）

### Requirement: 鬧鐘設定

使用者可以新增鬧鐘，設定鬧鐘時間與標籤。鬧鐘時間格式為 24 小時制，標籤用於識別不同鬧鐘。

#### Scenario: 使用者設定新鬧鐘
- **WHEN** 使用者輸入時間（時:分）並填寫標籤，點擊新增按鈕
- **THEN** 鬧鐘會被新增至鬧鐘列表，並顯示在列表中

#### Scenario: 使用者設定過去的時間
- **WHEN** 使用者設定的時間已過（當天早於當前時間）
- **THEN** 鬧鐘會自動設定為隔天的同一時間

### Requirement: 鬧鐘列表

系統顯示所有已設定的鬧鐘，列表中包含鬧鐘時間、標籤與啟用狀態。

#### Scenario: 顯示所有鬧鐘
- **WHEN** 使用者打開鬧鐘頁面
- **THEN** 系統顯示所有已設定的鬧鐘，排列順序為時間由近到遠

#### Scenario: 鬧鐘列表為空
- **WHEN** 使用者尚未設定任何鬧鐘
- **THEN** 系統顯示提示訊息「尚無鬧鐘設定」

### Requirement: 鬧鐘啟用/停用

使用者可以切換鬧鐘的啟用狀態。停用的鬧鐘不會觸發鬧鐘提醒。

#### Scenario: 停用鬧鐘
- **WHEN** 使用者點擊鬧鐘的停用按鈕
- **THEN** 鬧鐘狀態改為停用，不會在設定時間觸發

#### Scenario: 啟用鬧鐘
- **WHEN** 使用者點擊已停用鬧鐘的啟用按鈕
- **THEN** 鬧鐘狀態改為啟用，會在設定時間觸發

### Requirement: 刪除鬧鐘

使用者可以刪除已設定的鬧鐘。

#### Scenario: 刪除單一鬧鐘
- **WHEN** 使用者點擊鬧鐘的刪除按鈕
- **THEN** 該鬧鐘從列表中移除，且 localStorage 中的資料同步更新

### Requirement: 鬧鐘觸發

當鬧鐘時間到達時，系統顯示鬧鐘通知或鬧鐘介面。

#### Scenario: 鬧鐘時間到達
- **WHEN** 系統時間等於鬧鐘設定時間
- **THEN** 系統顯示鬧鐘提示（Alert 或 Modal），並播放音效

#### Scenario: 鬧鐘提示出現後
- **WHEN** 使用者點擊鬧鐘提示的關閉按鈕
- **THEN** 鬧鐘提示關閉，該鬧鐘自動改為停用狀態
