# 設計：add-calendar

## 背景

在現有的 React + Vite 專案中新增行事曆功能。這是一個全新的能力，不影響現有功能。

## 目標 / 非目標

### 目標
- 月檢視行事曆元件
- 事件 CRUD 操作
- 事件詳情檢視

### 非目標
- 不支援多日事件
- 不支援重複事件
- 不支援拖曳排程
- 不後端儲存（僅本地狀態）

## 決策

### 決策 1：日曆函式庫選擇

**選擇**：使用原生 JavaScript 計算日曆格，不引入外部日曆函式庫。

**理由**：
- 保持輕量，避免大型依賴
- 標準月檢視邏輯簡單，自行實作可控性更高
- 可根據需求靈活調整

**替代方案**：
- FullCalendar：功能完整但過於龐大
- react-big-calendar：需額外設定

### 決策 2：狀態管理

**選擇**：使用 React Context + useReducer 管理事件狀態。

**理由**：
- 事件狀態需跨元件共享
- 使用 Context避免 Prop Drilling
- useReducer 提供集中式狀態更新邏輯

### 決策 3：儲存方式

**選擇**：使用 localStorage 持久化事件。

**理由**：
- 無後端需求
- 簡單實作即可滿足需求
- localStorage 在瀏覽器重啟後仍保留資料

## 可測試性考量

- Calendar 元件可單元測試（渲染、導航）
- useEvents hook 可隔離測試（mock localStorage）
- EventModal 可整合測試

## 風險 / 取捨

- [風險] localStorage 容量限制（5MB）→ 監控事件數量，必要時提示用戶
- [取捨] 無後端同步 → 多裝置不同步（可接受）
