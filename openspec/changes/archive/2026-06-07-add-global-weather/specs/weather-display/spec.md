## 新增需求（ADDED Requirements）

### Requirement: 即時天氣資料取得

系統應（SHALL）透過 Open-Meteo API 取得各城市的即時天氣資料，包含溫度、相對濕度、風速及天氣狀態代碼。

#### Scenario: 成功取得單一城市天氣
- **WHEN** 系統使用有效的經緯度座標呼叫 Open-Meteo API
- **THEN** 系統應回傳該城市的溫度（°C）、濕度（%）、風速（km/h）及 WMO 天氣代碼

#### Scenario: API 回應失敗
- **WHEN** Open-Meteo API 回應非 200 狀態碼或網路錯誤
- **THEN** 系統應將該城市標記為載入失敗，顯示錯誤提示，不影響其他城市的資料顯示

#### Scenario: 批次取得多城市天氣
- **WHEN** 系統同時請求多個城市的天氣資料
- **THEN** 系統應使用 `Promise.allSettled` 批次處理，任一城市失敗不影響其他城市的資料

### Requirement: 天氣卡片展示

系統應（SHALL）以卡片形式展示每個城市的天氣資訊，包含城市名稱、國家、溫度、天氣描述圖示、濕度及風速。

#### Scenario: 正常顯示天氣卡片
- **WHEN** 天氣資料成功載入
- **THEN** 卡片應顯示城市名稱、國家名稱、溫度（依目前單位設定）、天氣圖示（依 WMO 代碼對應）、濕度百分比、風速

#### Scenario: 載入中狀態
- **WHEN** 天氣資料正在載入
- **THEN** 卡片應顯示載入指示器（如旋轉動畫或骨架屏）

#### Scenario: 載入失敗狀態
- **WHEN** 天氣資料載入失敗
- **THEN** 卡片應顯示「無法取得天氣資料」的提示文字，並提供重新嘗試的按鈕

### Requirement: 天氣代碼轉描述

系統應（SHALL）將 WMO 天氣代碼對應為人類可讀的天氣描述文字與圖示。

#### Scenario: 常見天氣代碼對應
- **WHEN** API 回傳 WMO 天氣代碼（如 0 = 晴天、61 = 小雨、95 = 雷雨）
- **THEN** 系統應將代碼轉換為對應的中文描述與圖示符號

### Requirement: 頁面導航切換

系統應（SHALL）提供頁面切換機制，讓使用者在鬧鐘頁面與天氣頁面之間導航。

#### Scenario: 切換至天氣頁面
- **WHEN** 使用者點擊「天氣」導覽標籤
- **THEN** 系統應顯示天氣資訊頁面，隱藏鬧鐘頁面

#### Scenario: 切換回鬧鐘頁面
- **WHEN** 使用者點擊「鬧鐘」導覽標籤
- **THEN** 系統應顯示鬧鐘頁面，隱藏天氣頁面
